Tutorial Sequence

Follow this structured sequence to build a Papers, Please–style inspection game using MakeCode Arcade. Each step introduces one clear mechanic that you will later combine into a complete game.

  1. STEP 01 Game Space & Player Control Click to expand

    Set up the game space and give the player control of a character. This defines how the player moves and interacts with the game world.

    Skills: Sprites · Controller input · Tilemaps

    Video

    TEACHER NOTES Hidden by default

    Focus: Movement should feel predictable.

    • Common misconception: forgetting collision walls.
    • Quick fix: enable tilemap walls early.
    • Extension: adjust speed for smoother control.

    Build Steps

    1. Create a player sprite.
    2. Add controller movement.
    3. Design a simple game space.
    4. Add walls or boundaries.
  2. STEP 02 Objects & Interaction Click to expand

    Introduce objects the player can interact with. These become the items inspected in the game.

    Skills: Sprites · Overlap events · Triggers

    Video

    TEACHER NOTES Hidden by default

    Focus: One interaction only.

    • Common misconception: adding too many triggers.
    • Quick fix: limit to one object type.
    • Extension: visual feedback on interaction.

    Build Steps

    1. Create an object sprite.
    2. Detect overlap with the player.
    3. Trigger an action or message.
  3. STEP 03 Information on Screen Click to expand

    Display information clearly so the player understands what they are inspecting.

    Skills: Text · Dialogue · Variables

    Video

    TEACHER NOTES Hidden by default

    Focus: Readability over style.

    • Common misconception: text appears too briefly.
    • Quick fix: pause gameplay when text appears.
    • Extension: button to dismiss text.

    Build Steps

    1. Show text when an object is activated.
    2. Store text in a variable.
    3. Control when text appears.
  4. STEP 04 Rules & Conditions Click to expand

    Add rules that check information and respond differently depending on conditions.

    Skills: Variables · If / else logic

    Video

    TEACHER NOTES Hidden by default

    Focus: One condition only.

    • Common misconception: stacking multiple rules.
    • Quick fix: test with two clear outcomes.
    • Extension: show rule result visually.

    Build Steps

    1. Create a rule using an if statement.
    2. Check a value.
    3. Trigger outcomes.
  5. STEP 05 Player Choices Click to expand

    Give the player choices and let those decisions affect what happens next.

    Skills: Controller buttons · Branching logic

    Video

    TEACHER NOTES Hidden by default

    Focus: Immediate feedback.

    • Common misconception: silent outcomes.
    • Quick fix: always show a response message.
    • Extension: sound or animation feedback.

    Build Steps

    1. Assign buttons to actions.
    2. Respond differently to choices.
    3. Give feedback.
  6. STEP 06 Game Flow & Progression Click to expand

    Control how the game progresses from one inspection to the next.

    Skills: Counters · States · Sequencing

    Video

    TEACHER NOTES Hidden by default

    Focus: Prevent skipping.

    • Common misconception: actions trigger twice.
    • Quick fix: disable inputs between states.
    • Extension: visible progress indicator.

    Build Steps

    1. Track progress with a counter.
    2. Move to the next item.
    3. Prevent actions out of order.
  7. STEP 07 End State & Reflection Click to expand

    Finish the game with a clear ending that explains outcomes and encourages reflection.

    Skills: End screens · Summary messages

    Video

    TEACHER NOTES Hidden by default

    Focus: Explanation over score.

    • Common misconception: ending too abrupt.
    • Quick fix: recap decisions made.
    • Extension: replay option.

    Build Steps

    1. Detect the end condition.
    2. Show a summary message.
    3. Explain decisions clearly.
