Tutorial Sequence

Follow this structured sequence to build a Papers, Please–style inspection game using MakeCode Arcade. Each step introduces one clear mechanic that you will later combine into a complete game.

  1. STEP 01 Game Space & Player Control Click to expand

    Set up the game space and give the player control of a character. This defines how the player moves and interacts with the game world.

    Skills: Sprites · Controller input · Tilemaps

    Video

    Build Steps

    1. Create a player sprite.
    2. Add movement using the controller.
    3. Design a simple game space.
    4. Add walls or boundaries.
  2. STEP 02 Objects & Interaction Click to expand

    Introduce objects the player can interact with. These will later become the core items used in the game.

    Skills: Sprites · Overlap events · Triggers

    Video

    Build Steps

    1. Create an object sprite.
    2. Detect when the player touches it.
    3. Trigger an action or message.
  3. STEP 03 Information on Screen Click to expand

    Display information clearly on screen so the player knows what is happening and what they are deciding.

    Skills: Text · Dialogue · Variables

    Video

    Build Steps

    1. Show text when an object is activated.
    2. Store text in a variable.
    3. Control when text appears and disappears.
  4. STEP 04 Rules & Conditions Click to expand

    Add rules that check what is happening in the game and respond differently depending on conditions.

    Skills: Variables · If / else logic

    Video

    Build Steps

    1. Create a rule using an if statement.
    2. Check a value or condition.
    3. Trigger different outcomes.
  5. STEP 05 Player Choices Click to expand

    Give the player clear choices and let those choices affect what happens next in the game.

    Skills: Controller buttons · Branching outcomes

    Video

    Build Steps

    1. Assign buttons to actions.
    2. Respond differently to each choice.
    3. Give feedback for each decision.
  6. STEP 06 Game Flow & Progression Click to expand

    Control how the game moves from one moment to the next so it plays smoothly and predictably.

    Skills: Counters · Game states · Sequencing

    Video

    Build Steps

    1. Track progress with a counter.
    2. Move to the next stage after a choice.
    3. Prevent actions happening out of order.
  7. STEP 07 End State & Reflection Click to expand

    Finish the game with a clear ending that explains what happened and encourages reflection on the player’s decisions.

    Skills: End screens · Summary messages

    Video

    Build Steps

    1. Detect when the game should end.
    2. Show a summary message.
    3. Explain outcomes clearly.
