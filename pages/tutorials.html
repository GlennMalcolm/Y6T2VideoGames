Tutorial Sequence

Follow this structured sequence to build a Papers, Please–style inspection game using MakeCode Arcade. Each step introduces one clear mechanic that you will later combine into a complete game.

  1. STEP 01 Inspector & Office Click to expand

    Create your inspector character and a simple office or checkpoint setting. Add basic movement so the inspector can move between areas such as the story desk and the approval stamp.

    Skills: Sprites · Controller movement · Basic tilemaps

    Build Steps

    1. Create an inspector sprite.
    2. Add controller movement (left/right/up/down).
    3. Build a simple tilemap (desk, stamp area, exit).
    4. Add collision walls to stop walking through desks.
    5. Test movement and adjust speed.

    Checkpoint

    • The inspector moves smoothly.
    • The office feels like a “work station”.
  2. STEP 02 Story Cards & Dialogue Click to expand

    Display incoming “story cards” based on headlines or posts. Use dialogue or on-screen text so players can read the story before making a decision.

    Skills: Overlap events · On-screen text · Simple dialogue

    Build Steps

    1. Create a “story card” sprite (paper icon).
    2. When the inspector overlaps the card, show text/dialogue.
    3. Store the current story text in a variable.
    4. Add a “next story” trigger (button press).

    Checkpoint

    • The player can reliably read the story.
    • The story appears only when triggered.
  3. STEP 03 Rubric Checks Click to expand

    Turn selected rubric questions into game logic. For example, if a story uses extreme emotional language or missing information, increase a manipulation score.

    Skills: Variables · If / else logic · Score tracking

    Build Steps

    1. Create a variable: manipulationScore.
    2. When a story is shown, set the score to 0.
    3. Add 1–3 checks (e.g., urgency, missing info, outrage).
    4. Increase the score when checks are present.

    Checkpoint

    • Score changes based on the story type.
    • The rules match the rubric language.
  4. STEP 04 Approve / Flag Decisions Click to expand

    Add player controls to APPROVE or FLAG a story. Link the outcome to the rubric logic so good decisions are rewarded and weak decisions are challenged.

    Skills: Controller input · Feedback text · Sound effects

    Build Steps

    1. Choose buttons (e.g., A=Approve, B=Flag).
    2. If score is high and player approves, show “Incorrect”.
    3. If score is high and player flags, show “Correct”.
    4. Add a stamp sound and a brief animation.

    Checkpoint

    • Every decision produces feedback.
    • The player understands why it was correct/incorrect.
  5. STEP 05 Multiple Stories & Endings Click to expand

    Create several inspection rounds. End the game with a summary that comments on how carefully the player applied the rules, not just how fast they finished.

    Skills: Arrays or lists · Game states · Multiple endings

    Build Steps

    1. Create a list/array of story texts.
    2. Track the current story index.
    3. After each decision, move to the next story.
    4. At the end, show a results screen with comments.

    Checkpoint

    • Game plays multiple rounds without breaking.
    • Ending explains performance clearly.
