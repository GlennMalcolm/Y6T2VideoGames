Tutorial Sequence

Follow this structured sequence to build a Papers, Please–style inspection game using MakeCode Arcade. Each step introduces one clear mechanic that you will later combine into a complete game.

  1. STEP 01

    Inspector & Office

    Create your inspector character and a simple office or checkpoint setting. Add basic movement so the inspector can move between areas such as the story desk and the approval stamp.

    Skills: Sprites · Controller movement · Basic tilemaps

  2. STEP 02

    Story Cards & Dialogue

    Display incoming “story cards” based on headlines or posts. Use dialogue or on-screen text so players can read the story before making a decision.

    Skills: Overlap events · On-screen text · Simple dialogue

  3. STEP 03

    Rubric Checks

    Turn selected rubric questions into game logic. For example, if a story uses extreme emotional language or missing information, increase a manipulation score.

    Skills: Variables · If / else logic · Score tracking

  4. STEP 04

    Approve / Flag Decisions

    Add player controls to APPROVE or FLAG a story. Link the outcome to the rubric logic so good decisions are rewarded and weak decisions are challenged.

    Skills: Controller input · Feedback text · Sound effects

  5. STEP 05

    Multiple Stories & Endings

    Create several inspection rounds. End the game with a summary that comments on how carefully the player applied the rules, not just how fast they finished.

    Skills: Arrays or lists · Game states · Multiple endings
